# 需求摘要与验收清单

本文整理自 `E:\EDGE\OA管理系统开发提示词.docx`，用于项目级验收和多代理协作对齐。

## 总体验收口径

- [ ] 后端使用 Spring Boot 3、Java 17、Maven。
- [ ] 前端使用 Vue 3、Vite、TypeScript、Element Plus。
- [ ] 后端采用 RESTful JSON API，所有接口统一 `/api` 前缀。
- [ ] 所有接口返回统一 `Result<T>`，包含 `code`、`message`、`data`、`timestamp`。
- [ ] 成功为 `200`，业务失败为 `400`，未认证为 `401`，无权限为 `403`，系统异常为 `500`。
- [ ] 使用 Spring Security + JWT，不使用 Session 作为主登录状态。
- [ ] 所有核心模块使用 MyBatis-Plus 完成规范化 CRUD。
- [ ] 所有核心业务表包含 `created_at`、`updated_at`、`deleted`。
- [ ] `deleted` 使用逻辑删除：`0` 未删除，`1` 已删除。
- [ ] 所有主键使用 `BIGINT AUTO_INCREMENT`。
- [ ] 所有列表支持分页、筛选和条件查询。
- [ ] Redis 缓存高频数据。
- [ ] RabbitMQ 负责异步通知事件。
- [ ] 站内信必须落库，在线用户通过 WebSocket 实时推送。
- [ ] 文件上传使用本地磁盘并保存元数据。
- [ ] AI 默认 mock，实现通义千问接入预留。
- [ ] 前端页面美观、现代、统一，不做简陋课程 Demo。

## 核心数据表

- [ ] `sys_user`：用户、角色、状态、部门、审批人身份。
- [ ] `sys_department`：部门树、负责人、默认审批人。
- [ ] `oa_approval`：审批主表。
- [ ] `oa_approval_record`：审批流转记录。
- [ ] `oa_schedule`：日程主表。
- [ ] `oa_schedule_participant`：会议日程参与人。
- [ ] `oa_news`：新闻主表。
- [ ] `oa_news_comment`：新闻评论。
- [ ] `oa_news_like`：新闻点赞。
- [ ] `oa_news_favorite`：新闻收藏。
- [ ] `sys_dict_type`：字典类型。
- [ ] `sys_dict_data`：字典数据。
- [ ] `sys_file`：文件元数据。

- [ ] `sys_ai_log`：AI 调用日志。

## 角色与权限

### 管理员 `ADMIN`

- [ ] 可以登录系统。
- [ ] 可以管理用户、部门、字典、文件。
- [ ] 可以设置或取消员工审批人身份。
- [ ] 可以管理全部审批单并审批任何审批单。
- [ ] 可以管理全部日程。
- [ ] 可以管理新闻、评论、点赞和收藏数据。
- [ ] 可以查看和发送系统通知。
- [ ] 可以使用全部 AI 功能。
- [ ] 可以查看系统统计数据。

### 员工 `EMPLOYEE`

- [ ] 可以登录系统。
- [ ] 可以查看和修改个人资料。
- [ ] 可以查看部门信息。
- [ ] 可以发起审批。
- [ ] 可以查看自己发起的审批。
- [ ] 员工被设置 `is_approver = 1` 后，可以查看并处理待自己审批的审批。
- [ ] 可以查看自己已审批的审批。
- [ ] 可以管理个人日程，参与会议日程。
- [ ] 可以查看新闻、评论新闻、点赞新闻、收藏新闻。
- [ ] 可以查看自己的站内信。
- [ ] 可以使用智能问答助手和日程 AI 助手。
- [ ] 不可使用审批 AI 摘要、审批风险提示、新闻 AI 生成、新闻 AI 润色。

### 审批权限

- [ ] 可审批人员为管理员或 `is_approver = 1` 的员工。
- [ ] 普通员工不能审批。
- [ ] 审批采用单级审批。
- [ ] 员工提交审批时不能任意选择审批人。
- [ ] 系统根据部门默认审批人或部门负责人自动匹配审批人。

## 用户模块

- [ ] 用户分页查询。
- [ ] 支持按用户名、姓名、手机号、部门、状态筛选。
- [ ] 新增、修改、逻辑删除、批量删除员工。
- [ ] 启用和禁用员工。
- [ ] 重置密码、修改密码。
- [ ] 上传头像。
- [ ] 设置员工是否为审批人。
- [ ] 查询审批人列表。
- [ ] 查询和修改当前登录用户资料。
- [ ] 密码使用 BCrypt 加密。
- [ ] 用户登录账号不能重复。
- [ ] 用户禁用后不能登录。
- [ ] 管理员不能删除自己。
- [ ] 员工只能修改自己的基础资料，不能修改角色、状态、审批人身份。

## 部门模块

- [ ] 部门树查询。
- [ ] 部门分页查询。
- [ ] 新增、修改、逻辑删除、批量删除部门。
- [ ] 启用和禁用部门。
- [ ] 设置部门负责人。
- [ ] 设置部门默认审批人。
- [ ] 查询部门员工列表。
- [ ] 删除部门前检查是否存在子部门或员工。
- [ ] 部门默认审批人必须是管理员或 `is_approver = 1` 的员工。
- [ ] 部门树使用 Redis 缓存。

## 审批模块

- [ ] 审批类型支持 `LEAVE`、`REIMBURSEMENT`、`OVERTIME`、`TRAVEL`。
- [ ] 审批状态支持 `DRAFT`、`PENDING`、`APPROVED`、`REJECTED`、`WITHDRAWN`。
- [ ] 新建、修改、删除审批草稿。
- [ ] 提交审批、撤回审批、审批通过、审批驳回。
- [ ] 查询我的申请、待我审批、我已审批。
- [ ] 管理员可查询全部审批。
- [ ] 审批详情和审批记录查询。
- [ ] 支持上传审批附件。
- [ ] 管理员可触发 AI 摘要和 AI 风险提示。
- [ ] 草稿可以修改，已提交审批不能随意修改。
- [ ] 待审批状态可以撤回。
- [ ] 提交时根据部门负责人或部门默认审批人自动确定审批人。
- [ ] 提交、通过、驳回等事件进入 RabbitMQ，再生成站内信并推送 WebSocket。

## 日程模块

- [ ] 日程类型支持 `PERSONAL` 和 `MEETING`。
- [ ] 新增、修改、删除、批量删除日程。
- [ ] 日程详情、分页查询、今日日程、本周日程、日历视图查询。
- [ ] 创建会议日程。
- [ ] 添加和移除参与人。
- [ ] 参与人接受或拒绝会议。
- [ ] 支持上传日程附件。
- [ ] 支持提前 5 分钟、15 分钟、30 分钟、1 小时提醒。
- [ ] 使用 Spring Scheduled 扫描待提醒日程。
- [ ] 到达提醒时间后发送 RabbitMQ 消息。
- [ ] 消费者写入站内信、发送 WebSocket 推送、发送邮件提醒。
- [ ] 邮件发送失败不影响主流程。
- [ ] 日程 AI 助手可将自然语言解析为标题、开始时间、结束时间、地点、参与人、提醒时间、日程类型。

- [ ] 新闻分页查询和详情。
- [ ] 新增、修改、删除、批量删除新闻。
- [ ] 草稿保存、发布、下架、置顶和取消置顶。
- [ ] 上传封面图和新闻附件。
- [ ] 增加阅读量。
- [ ] 评论、删除评论。
- [ ] 点赞和取消点赞。
- [ ] 收藏和取消收藏。
- [ ] 查询我的收藏。
- [ ] 管理员管理全部评论。
- [ ] 管理员可使用新闻 AI 生成和新闻 AI 润色。
- [ ] 员工只能查看已发布新闻。
- [ ] 员工评论直接显示，不做审核。

- [ ] 新闻详情使用 Redis 缓存。

## 字典模块

- [ ] 字典类型 CRUD。
- [ ] 字典数据 CRUD。
- [ ] 字典数据启用和禁用。
- [ ] 根据字典类型编码查询字典项。
- [ ] 批量删除字典。
- [ ] 刷新字典缓存。
- [ ] 预置用户状态、用户角色、部门状态、审批类型、审批状态、日程类型、日程提醒方式、新闻分类、新闻状态、通知类型、AI 功能类型。
- [ ] 查询字典优先读 Redis。
- [ ] 新增、修改、删除字典后清理或刷新 Redis 缓存。

## 文件模块

- [ ] 文件上传、下载、详情、列表、删除。
- [ ] 根据业务类型和业务 ID 查询附件。
- [ ] 支持头像、审批附件、日程附件、新闻封面图、新闻附件。
- [ ] 使用本地磁盘存储。
- [ ] 按年月分目录存储。
- [ ] 使用 UUID 重命名文件。
- [ ] 保存文件元数据到 `sys_file`。
- [ ] 图片缩略图能力可预留。
- [ ] 文件上传和大文件处理可以异步。
- [ ] 文件记录逻辑删除，物理删除可异步执行。

## 通知、RabbitMQ 与 WebSocket

- [ ] 通知链路为业务事件 -> RabbitMQ -> 消费者 -> 站内信落库 -> WebSocket 实时推送 -> 可选邮件提醒。
- [ ] 查询我的站内信。
- [ ] 查询未读数量。
- [ ] 标记单条已读。
- [ ] 批量标记已读。
- [ ] 删除通知。
- [ ] 管理员发送系统通知。
- [ ] WebSocket 推送事件包含新审批待处理、审批通过或驳回、新日程提醒、会议邀请、新闻发布、新闻评论、系统通知。
- [ ] RabbitMQ 使用 TopicExchange。
- [ ] 消息使用 JSON 序列化。
- [ ] 开启生产者确认和 returns callback。
- [ ] 消费者使用手动 ACK。
- [ ] 消费失败记录日志，避免无限重试。
- [ ] WebSocket 地址为 `/ws/notification?token=xxx`。
- [ ] 建立连接时验证 JWT。
- [ ] 用户离线时只保存站内信，不报错。

- [ ] 审批风险提示输出风险等级、风险原因、建议处理意见。
- [ ] 新闻 AI 生成输入主题、关键词、语气、字数、分类，输出标题、摘要、正文。
- [ ] 新闻 AI 润色输入原标题、原正文、润色风格，输出润色后标题、摘要、正文。
- [ ] 日程 AI 助手输入自然语言，输出结构化日程。
- [ ] 智能问答助手覆盖审批流程、请假规则、报销规则、加班规则、日程使用说明、新闻查看说明、字典配置说明。
- [ ] 所有 AI 调用必须记录 `sys_ai_log`。

建议 key：

```text
dict:type:{typeCode}
department:tree
news:detail:{id}
news:list:published
user:permission:{userId}
notification:unread:{userId}
```
